⚠ JavaScript Drama

Programming is not magic. Humans think in big ideas. Computers think in tiny exact steps. That is why programming languages exist. They translate our ideas into instructions a computer can follow.

The First Line

const checkboxes =
document.querySelectorAll(".checklist input");

Imagine asking someone: "Please remember every checkbox on this page." JavaScript cannot understand that sentence directly. Instead it uses its own grammar.

const means: Create a container with a name. That container will be called checkboxes.

document means: "The webpage itself." Browsers think every webpage is one large document.

query means: Ask.

selector means: How do we recognise something?

All means: Don't stop at one. Find every matching checkbox.

Reading the whole sentence like English: 


Create a container

called checkboxes.

Go to the webpage.

Find every checkbox.

Put them into that container.

Then JavaScript waits. 


box.addEventListener("change", function(){

});

The computer is not constantly checking. It quietly waits.

addEventListener means "When this thing changes... tell me."

Nothing happens... until a human clicks.

Then comes the question. 


if(checked===total){

}

Humans say "When everything is finished..."

JavaScript asks "Are these two numbers exactly equal?"

if means Ask a question.

=== means Exactly equal.

Why block? 


message.style.display="block";

This line surprises almost everyone. JavaScript is borrowing a word from CSS.

Earlier, CSS hid the message using display:none;

JavaScript simply tells CSS: 


Please stop hiding it.

Display it as a normal block.

JavaScript doesn't decide what "block" means. CSS already knows. JavaScript just asks CSS to use that style.

The Big Picture

HTML creates things.

CSS dresses things.

JavaScript watches people.

When people do something... JavaScript changes the page.

🎨 CSS Academy

HTML builds the house. CSS decorates it.

Why a separate language?

Imagine writing a novel. Would you write the story and the printing instructions inside the same sentence? Probably not.

HTML describes WHAT exists. CSS describes HOW it should look.

display:block

Think of a sheet of paper. A block starts on its own line and occupies the available width. Paragraphs, cards and sections are normally blocks.

Classes

A class is like giving several people the same coloured badge.

Anyone wearing the "done" badge gets crossed out.

IDs

An ID is different. Only ONE element should have it.

Like your passport number.
